# Security policy

Do not report exposed credentials in a public issue. Revoke or rotate the credential first, then contact the maintainer privately through the contact method listed on the GitHub profile.

Useful reports include:

- the affected version or commit;
- a minimal reproduction that does not contain a live secret;
- expected and actual behaviour;
- the practical impact;
- a suggested fix, when available.

`repo-guard` performs heuristic local scanning. A clean result is not a guarantee that a repository contains no sensitive data.
